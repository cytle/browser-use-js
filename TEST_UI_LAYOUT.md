# 测试UI布局说明

## 概述

测试UI现在采用左右分栏的固定布局设计，为Browser-Use JS的测试和演示提供更好的用户体验。

## 布局结构

### 左侧区域 - 测试页面内容

- **位置**: 屏幕左侧，占据大部分空间
- **功能**: 显示被测试的网页内容
- **内容**:
  - 模拟的电商网站页面 (TechStore)
  - 包含搜索、购物车、产品展示等交互元素
  - 用于演示AI代理的网页交互能力

### 右侧区域 - 测试控制面板

- **位置**: 屏幕右侧，固定宽度 (384px)
- **功能**: 测试控制和监控界面
- **特性**:
  - 可折叠/展开 (点击箭头按钮)
  - 折叠时只显示控制按钮
  - 展开时显示完整的测试界面

## 测试功能模块

右侧面板包含以下测试模块：

1. **综合演示** - 完整的Browser-Use JS工作流程演示
2. **系统测试** - 系统初始化和配置测试
3. **DOM处理** - 页面元素识别和交互测试
4. **AI代理** - AI代理任务理解和执行测试
5. **性能监控** - 系统性能和资源使用监控
6. **日志查看** - 系统运行日志和调试信息

## 使用方法

1. **启动开发服务器**:

   ```bash
   pnpm dev
   ```

2. **访问测试界面**:

   - 打开浏览器访问 `http://localhost:3000`
   - 左侧显示测试页面内容
   - 右侧显示测试控制面板

3. **交互测试**:
   - 在左侧页面进行各种交互操作
   - 在右侧面板选择相应的测试模块
   - 观察AI代理的行为和系统响应

## 设计优势

1. **分离关注点**: 测试内容和控制界面分离，互不干扰
2. **空间利用**: 左侧大空间用于显示测试页面，右侧紧凑空间用于控制
3. **用户体验**: 固定布局避免界面跳动，提供稳定的测试环境
4. **可扩展性**: 左侧可以轻松替换为其他测试页面或iframe内容

## 技术实现

- **框架**: React 19 + TypeScript
- **样式**: Tailwind CSS 4.x
- **组件库**: Radix UI (Shadcn)
- **图标**: Lucide React
- **状态管理**: React Hooks + Zustand

## 自定义测试页面

如需替换左侧的测试页面内容，可以：

1. 修改 `TestPageContent.tsx` 组件
2. 或者创建新的组件并在 `TestApp.tsx` 中引用
3. 也可以使用iframe嵌入外部页面

## 响应式设计

- 在较小屏幕上，右侧面板会自动调整宽度
- 支持折叠功能以节省空间
- 保持核心功能在各种屏幕尺寸下的可用性
